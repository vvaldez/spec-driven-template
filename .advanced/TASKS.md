# TASKS.md (optional advanced template)

## Purpose
Use this when you need a **phased checklist** derived from your spec (similar in spirit to [GitHub Spec Kit’s tasks template](https://github.com/github/spec-kit/blob/main/templates/tasks-template.md))—not for Persona 1’s first day.

- **Persona 1 minimum:** `SPEC.md` (outcomes) + gitignored `CONTEXT.md` + `CONTEXT_TRANSFER` pattern; optionally **`TASKS.md`** at repo root from `TASKS.example.md` (also gitignored) for a short personal checklist.
- **`MILESTONES.md`:** dated **timeline** checkpoints; **this file** is for **ordered work units**.

## How to use
1. Pull tasks from **acceptance criteria** and **High-Level Feature List** in `SPEC.md`.
2. Group by phase or user story; keep each line **one concrete action**.
3. Prefer **checklists** (`- [ ]`) so you can hand off a bounded file to another model or device.

## Phase: Setup
- [ ] _(example: confirm vision section in SPEC.md is stable enough to slice work)_

## Phase: Foundational (blocking)
- [ ] _(example: agree on naming and file layout in planning docs)_

## Phase: Delivery slices
- [ ] _(tie each item to a SPEC feature or acceptance criterion)_

## Notes
- For **upstream template work**, root **`TODO.md`** is **maintainer milestones**—not this file.
- Do not put **secrets** in task text; see `SECURITY.md`.
